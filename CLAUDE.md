# プロジェクト概要
- 目的: 毎日の日報作成を半自動化し、過去のデータから文体を学習するローカルCLIツールの開発。
- アーキテクチャ: Teamsチャットのメモとカレンダーの予定を取得し、過去の日報履歴をコンテキストとしてLLMに渡し、パーソナライズされた日報を生成する。

# 技術スタック
- 言語: TypeScript (Node.js)
- 実行環境: ターミナルから実行するCUIツール (`ts-node` 等)
- データ保存: ローカルのJSONファイル (`history.json`) を簡易データベースとして使用

# コーディングルール
- 常に型（Interface/Type）を明示的に定義すること。

# セキュリティ・ポリシー（最重要）
- 認証情報 (APIキー) は `.env` で管理し、絶対にハードコードしない。
- `history.json` や `.env` は必ず `.gitignore` に含め、外部にコミットしないこと。
- 開発初期（ハーネス構築時）は、外部APIとは通信せず、コード内のダミーデータを使用すること。
